В Qlik Sense “из коробки” доступно порядка трех десятков коннекторов к различным сервисам и базам данных. Почти всем известно, что существуют дополнительные пакеты коннекторов? распространяемые отдельно самой Qlik Tech и/или сторонними разработчиками. Как раз таким пакетом является Qlik Web Connectors который содержит дополнительные коннекторы, которые можно использовать из Qlik Sense Enterprise или Qlik View.

У вас есть право использовать Qlik Web Connectors если у вас есть лицензия, например, на Qlik Sense Enterprise. Сейчас Qlik Web Connectors это бандл, в который входит около 40 бесплатных коннекторов. С недавнего времени правила лицензирования этого бандла поменялись, а раньше часть Premium-коннекторов была платная.

Как минимум два коннектора мы активно используем для своих внутренних задач:

  • Qlik Google Drive & Spreadsheets Connector: некоторые аналитические приложения генерируют и загружают данные хранящиеся на Google Drive
  • Qlik JIRA Connector используем для генерации отчетов на основании данных Atlassian Jira

После установки Qlik Web Connectors работает как web-сервис, по сути являясь промежуточным слоем загрузки данных из источников. В момент загрузки данных используются открытые API сервисов, а со стороны приложения Qlik выполняется стандартный скрипт загрузки отправляющий запросы в Qlik Web Connectors.

Установка

1. Получение дистрибутива

Подразумевается, что у вас есть доступ к порталу загрузки продуктов Qlik и вы уже скачали zip файл. Например он может называться как-то так: QlikWebConnectorsDecember2020Patch2.zip. Пользователи Qlik Sense или Qlik View с активной подпиской/поддержкой могут перейти на портал загрузки по кнопке Download Qlik Product в правом верхнем углу сайта.

2. Разблокируйте zip-файл

Этап который не стоит пропускать, но некоторые склонны его игнорировать! После загрузки ZIP-файла необходимо установить чек-бокс “Разблокировать”

3. Распакуйте архив в папку

В моем примере: c:\WebConnector

4. Определяемся со сценарием использования

На этом этапе вы можете начать использовать коннекторы если вы единственный пользователь и нет задачи настроить авторизацию и ограничить доступ к функционалу. В таком случае можете сразу переходить к последнему пункту.

В противном случае необходима дополнительная настройка.

5. Установка и запуск Web Connectors как Windows Service

Переходим в каталог BatchFiles:

Перед запуском необходимо проверить корректность путей в bat-файле. Откройте файл Install Service.bat в любом текстовом редакторе и убедитесь, что указаны правильные пути: Путь к фреймворку Microsoft.NET у вас может быть другой.

REM *****************************
REM IMPORTANT - PLEASE READ:
REM * You should ensure than you have 'unblocked' the Qlik Web Connectors download zip file BEFORE unzipping it and running this.
REM * You need to change the following paths to match your .NET installation directory and QlikWebConnectorsSVC.exe location.
REM * You should run this file as Administrator (right click, 'Run as administrator').
REM *****************************
"%windir%\Microsoft.NET\Framework\v4.0.30319\installutil" "c:\WebConnector\QlikWebConnectorsSVC.exe"

pause

Возможна у вас уже был установлен этот сервис с другими путями или вы хотите переустановить службу. Тогда запустите оснастку “services.msc” и если видите в списке сервисов упоминание Qlik Web Connectors, то выполните от имени Администратора указанную команду:

Запускаем файл Install Service.bat в режиме Запуск от имени Администратора

Если на этом этапе возникли ошибки, то, как правило, вы или не разблокировали zip (шаг 2) или не исправили пути в bat-файле. Проверьте все еще раз.

6. Запуск сервиса

В оснастке “services.msc” у вас должна появиться служба Qlik Web Connectors. Запускаем правой кнопкой мыши:

Если сервис стартует и сразу останавивается, то мне известны две причины (их разумеется больше):

  • Имеется уже запущенная копия Web Connectors возможно запущенная вручную - проверьте задачи в трее и остановите работающий экземпляр.
  • Не достаточно прав для захвата 5555/tcp порта. В этом случае рекомендуется создать аккаунт и добавить его в группу Администраторы. После этого на вкладке сервиса “Вход в систему” указываем логин и пароль созданного аккаунта:

Подробности о неудачном запуске службы можно узнать в лог-файле Service_Startup_Log.txt, который находится в папке WebConnector.

7. Подключение к Qlik Web Connectors

На компьютере с установленным Qlik Web Connectors открываем браузер и переходим по адресу http://localhost:5555 Читаем и принимаем лицензионное соглашение и получаем доступ к административной консоли.
